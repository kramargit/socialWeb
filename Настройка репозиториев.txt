
Как подготовить, настроить удаленный репозиторий на GitHub:
Чтобы создать новый репозиторий на GitHub, вам необходимо выполнить следующие действия:

Войдите в свою учетную запись GitHub и нажмите значок «+» в правом верхнем 
углу страницы. В раскрывающемся списке выберите «Новый репозиторий».
На странице «Создать новый репозиторий» вам нужно будет выбрать имя для
 вашего репозитория и добавить описание (если хотите).
Затем выберите, будет ли ваш репозиторий общедоступным (видимым для всех в Интернете)
 или частным (видимым только вам и людям, которым вы предоставили доступ). 
Обратите внимание, что для частных репозиториев требуется платная подписка
 на GitHub.
Вы также можете добавить файл README в свой репозиторий, что является хорошей
 практикой для объяснения того, о чем ваш проект и как его использовать.
Наконец, нажмите кнопку «Создать репозиторий», 
чтобы завершить настройку нового удаленного репозитория на GitHub.





Как оформить мой код в этот репозиторий:
После того, как вы создали удаленный репозиторий на GitHub, вам нужно будет 
извлечь свой код из локального репозитория и отправить его в удаленный репозиторий. 
Вот как это сделать:

Во-первых, вам нужно инициализировать репозиторий Git в вашем локальном 
каталоге проекта, если вы еще этого не сделали. Для этого запустите команду 
«git init» в 
каталоге вашего проекта.
Затем вам нужно добавить файлы из каталога вашего проекта в репозиторий Git. 
Запустите команду «git add». чтобы добавить все файлы в вашем каталоге в репозиторий.
Теперь зафиксируйте изменения, которые вы внесли в свои файлы, с помощью команды 
«git commit -m 'initial commit'».
Далее вам нужно настроить удаленный репозиторий как удаленный репозиторий Git.
Для этого выполните команду «git remote add origin <URL-адрес удаленного репозитория>». 
Замените <URL-адрес удаленного репозитория> URL-адресом вашего удаленного 
репозитория на GitHub.
Наконец, отправьте свой код в удаленный репозиторий с помощью команды 
«git push -u origin master». Это отправит ваш код в основную ветку 
удаленного репозитория.



Чтобы загрузить файлы из удаленного репозитория GitHub во вновь созданный локальный
репозиторий, вы можете выполнить следующие шаги:


Создайте новый каталог на своем компьютере, где вы хотите хранить файлы репозитория.

Откройте терминал и перейдите во вновь созданный каталог с помощью команды cd.


Клонируйте удаленный репозиторий в локальный репозиторий с помощью команды git clone.
Например, если URL-адрес удаленного репозитория https://github.com/username/repo.git,
вы можете клонировать его в свой локальный репозиторий, выполнив следующую команду:

git clone https://github.com/username/repo.git

После того, как репозиторий будет клонирован, вы должны увидеть новый каталог с
имя репозитория, содержащее все файлы из удаленного репозитория.

Если вы хотите загружать только определенные файлы или папки из удаленного репозитория,
вы можете использовать команду git archive. Вот пример:

Создайте новый каталог на вашем компьютере, где вы хотите хранить загруженные файлы.

Откройте терминал и перейдите во вновь созданный каталог с помощью команды cd.

Загрузите нужные файлы или папки из удаленного репозитория, используя
Команда git архива. Например, если вы хотите загрузить папку src
из удаленного репозитория вы можете запустить следующую команду:

git archive --remote=https://github.com/username/repo.git HEAD src | tar xf -

Эта команда загрузит папку src и извлечет ее в ваш текущий каталог.



а если я хочу скачать данные с другой ветки?


Вы можете использовать следующую команду для загрузки определенной ветки
из удаленного репозитория:

git clone -b branch_name https://github.com/username/repo.git

Замените имя_ветви на имя ветки, которую вы хотите загрузить.
и https://github.com/username/repo.git с URL-адресом удаленного репозитория.
Эта команда создаст новый локальный репозиторий с содержимым файла
указанная ветка из удаленного репозитория.

В качестве альтернативы, если у вас уже есть локальный репозиторий,
 вы можете использовать следующую команду, чтобы вытащить определенную ветку из
 удаленный репозиторий:

 git pull origin branch_name

Эта команда обновит ваш локальный репозиторий содержимым файла
указанная ветка из удаленного репозитория.
